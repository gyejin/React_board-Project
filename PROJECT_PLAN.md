# React 게시판 프로젝트 개선 계획

이 문서는 React 게시판 프로젝트의 기능 추가 및 디자인 개선 계획을 정리하고 진행 상황을 추적하기 위해 작성되었습니다.

## 1단계: 핵심 기능 강화

- [x] **댓글 시스템 구현**
    - [x] **백엔드**
        - [x] `Comment` 엔티티 생성 (사용자, 게시물 관계 설정)
        - [x] 댓글 CRUD API 엔드포인트 구현 (`comments.controller.ts`)
        - [x] 댓글 작성자 권한 제어 (수정/삭제)
    - [x] **프런트엔드**
        - [x] 게시물 상세 페이지에 댓글 목록 표시 (`PostDetailPage.js`)
        - [x] 댓글 작성 폼 컴포넌트 생성 (`CommentForm.js`)
        - [x] 작성자에게만 수정/삭제 버튼 표시

- [x] **게시물 검색 기능**
    - [x] **백엔드:** 제목/내용 기반 검색 API 엔드포인트 추가 (`/posts/search`)
    - [x] **프런트엔드:** 게시물 목록 페이지에 검색창 추가 (`PostsPage.js`)

- [x] **게시물 페이지네이션**
    - [x] **백엔드:** 페이지네이션 API 구현 (`/posts?page=1&limit=10`)
    - [x] **프런트엔드:** 페이지 이동 UI 구현 (다음/이전 버튼)

- [x] **사용자 프로필 개선**
    - [x] **백엔드:** 사용자가 작성한 게시물 목록 API (`/users/:userId/posts`)
    - [x] **프런트엔드:** 마이페이지(`MyPage.js`)에서 작성한 게시물 목록 확인 기능

## 2단계: UI/UX 디자인 개선

- [ ] **UI 컴포넌트 라이브러리 통합**
    - [ ] `client` 프로젝트에 **DaisyUI** 설치
    - [ ] `tailwind.config.js`에 DaisyUI 플러그인 추가

- [ ] **레이아웃 및 컴포넌트 재디자인**
    - [ ] 공통 레이아웃 (`Navbar.js`, `Footer.js`) 제작
    - [ ] 주요 페이지 (메인, 게시물 목록, 로그인 등) DaisyUI 컴포넌트로 재디자인
    - [ ] 반응형 디자인 적용 (모바일, 태블릿, 데스크톱)
